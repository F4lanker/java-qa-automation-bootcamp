## Git Commit Rules

### Types
feat     — new feature or file
fix      — bug fix or review correction  
refactor — restructure without behavior change
test     — test additions or updates
docs     — documentation only
chore    — dependencies, config, build

### Rules
- One commit = one logical unit
- Always add scope: feat(util):, fix(schemas):, test(day26):
- No non-standard types (data:, update:, etc.)
- After review: use fix: or refactor: depending on type of change
